Getting Started

Prerequisites

  • AWS Account with appropriate permissions
  • Node.js 14.x or later
  • AWS CDK installed
  • Messaging platform configured (Teams, Slack, etc.)

Quick Setup

To begin using the AWS Bot, you'll need to configure your environment and deploy the necessary components. Follow these basic steps to get started:

  1. Clone the repository and navigate to the project directory
  2. Install dependencies:
    npm install
  3. Configure AWS credentials:
    aws configure
  4. Deploy the bot using CDK:
    npm run cdk deploy
  5. Configure your messaging platform webhook
Note: For detailed setup instructions, refer to the Setup Guide.

Custom Adaptive Cards

You can create custom Adaptive Cards for specific AWS services or event types. Place your custom card templates in the adaptiveCards directory and register them in the card registry.

// Add your custom card to the registry const cardTemplates = { 'notification-default': notificationDefault, 'generic-card': genericCard, 'ssm-card': ssmCard, 'your-custom-card': yourCustomCardTemplate };
Important: Make sure your custom card template follows the Adaptive Card schema and includes the necessary placeholder variables.

Configuration

Event Mapping

The bot maps AWS events to specific Adaptive Card templates. You can customize this mapping in the configuration file:

{ "eventMappings": { "aws.ec2": "ec2-card", "aws.ssm": "ssm-card", "aws.cloudwatch": "cloudwatch-card", "default": "notification-default" } }

Custom Settings

Troubleshooting

Common Issues

  • Notification not received

    Check Lambda execution logs and ensure the webhook URL is correctly configured.

  • Placeholder variables not replaced

    Verify that your event data structure matches the placeholders in your Adaptive Card template.

  • Card rendering incorrectly

    Validate your Adaptive Card JSON using the Adaptive Cards Designer.
